Анімації

Візуальне покращення елементів дизайну та створення незабутнього враження для відвідувачів.

Коротко про Анімації

За останні 10 років продуктивність браузерів значно зросла. Раніше було набагато складніше додавати інтерактивну анімацію на веб-сторінки через потенційні проблеми з рендерингом і продуктивністю. Але сьогодні CSS-анімації можна використовувати набагато вільніше, а синтаксис, як правило, набагато простіше вивчити порівняно з такими функціями CSS, як Flexbox або Grid Layout.

Існують різні функції, які є частиною специфікації анімації W3C CSS. Деякі з них простіші у використанні, ніж інші. У цьому підручнику з анімації ключових кадрів CSS ми розглянемо весь синтаксис, включаючи кожну з різних властивостей. Я додам інтерактивні демонстрації, які допоможуть вам зрозуміти, що можливо за допомогою CSS-анімації.

Синтаксис @keyframes

Кожна CSS-анімація складається з двох частин: одна або декілька властивостей animation-* разом з набором ключових кадрів анімації, які визначаються за допомогою правила @keyframes як правило. Давайте детально розглянемо, що входить до складу набору правил @keyframes.

Синтаксис виглядає наступним чином:

@keyframes moveObject {
    0% {
        transform: translateX(0);
    }

    50% {
        transform: translateX(300px);
    }

    100% {
        transform: translateX(300px) scale(1.5);
    }
}

Як показано тут, правило @keyframes складається з трьох основних частин:

  • Правило @keyframes, за яким слідує користувацьке ім'я анімації
  • Набір фігурних дужок, що обгортають усі ключові кадри
  • Один або кілька ключових кадрів, кожен з яких містить відсоток, за яким слідує набір правил, оточений фігурними дужками

У цьому першому прикладі нашого підручника з CSS-анімації я визначив свою анімацію з назвою moveObject. Це може бути будь-яка назва, чутлива до регістру, і вона має відповідати правилам створення користувацьких ідентифікаторів у CSS. Це користувацьке ім'я має збігатися з ім'ям, яке використовується у властивості animation-name (про яку ми поговоримо пізніше).

У моєму прикладі ви помітите, що я використовував відсотки для визначення кожного ключового кадру в анімації. Якщо моя анімація містить ключові кадри, які дорівнюють 0% і 100%, я можу альтернативно використовувати ключові слова from і to:

@keyframes moveObject {
    from {
        transform: translateX(0);
    }

    50% {
        transform: translateX(300px);
    }

    to {
        transform: translateX(300px) scale(1.5);
    }
}

Результат цього коду ви можете переглянути на Codepen

Дещо про синтаксис @keyframes:

  • Ви можете розмістити набори правил для ключових кадрів у будь-якому порядку, і браузер все одно буде виконувати їх у порядку, визначеному їхнім відсотковим співвідношенням
  • Ви можете опустити ключові кадри 0% і 100%, і браузер автоматично визначить їх за наявними стилями елемента, який анімується
  • Якщо є повторювані ключові кадри з різними наборами правил (наприклад, два ключові кадри для 20% з різним значенням трансформації), браузер використовує останній з них
  • Ви можете визначити кілька ключових кадрів в одному селекторі ключових кадрів через кому: 10%, 30% { ... }
  • Ключове слово !important обнуляє будь-яке значення властивості, тому його не слід використовувати для властивості всередині набору правил ключового кадру

animation-name

Як уже згадувалося, кожна CSS-анімація, яку ви створюєте, повинна мати ім'я, що з'являється в синтаксисі @keyframes. Це ім'я має збігатися з ім'ям, визначеним за допомогою властивості animation-name.

Використовуючи CSS з попередньої демонстрації, синтаксис виглядає наступним чином:

.box {
    animation-name: moveObject;
}

Знову ж таки, користувацьке ім'я, яке я визначив, має також існувати як ім'я для правила @keyframes - інакше це ім'я нічого не робитиме. Ви можете думати про це як про виклик функції у JavaScript. Сама функція буде частиною коду @keyframes moveObject {}, а виклик функції - animation-name: moveObject;.

Деякі речі, які слід пам'ятати про animation-name:

  • Початкове значення параметра animation-name - none, що означає, що жоден ключовий кадр не активний. Це можна використовувати як своєрідний "перемикач" для вимкнення анімації.
  • Вибране вами ім'я враховує регістр і може містити літери, цифри, підкреслення та дефіси.
  • Першим символом у назві має бути буква або дефіс, але тільки один дефіс.
  • Ім'я не може бути зарезервованим словом, таким як unset, initial або inherit.

animation-duration

Властивість animation-duration, як це не дивно, визначає кількість часу, необхідного для запуску анімації від початку до кінця. Це значення можна вказати у секундах або мілісекундах, як показано нижче:

.box {
    animation-duration: 2s;
}

Вищесказане було б еквівалентно наступному:

.box {
    animation-duration: 2000ms;
}

Ви можете побачити властивість animation-duration в дії в наступній демонстрації Codepen. У цій демонстрації ви можете вибрати, як довго триватиме анімація.

Якщо ви використовуєте мале число разом з ms для одиниці виміру, ви можете навіть не помітити жодного руху. Спробуйте встановити більше число, якщо ви використовуєте мілісекунди.

Кілька зауважень щодо використання animation-duration:

  • Від'ємні значення є неприпустимими
  • Пристрій має бути увімкнений, навіть якщо тривалість встановлена на 0s (початкове значення)
  • Ви можете використовувати дробові значення (наприклад, 0.8s)

animation-timing-function

Функція animation-timing-function, яка не настільки очевидна за своїм значенням, як попередні дві властивості, використовується для визначення способу виконання CSS-анімації. Можливо, це не найзрозуміліше пояснення, але синтаксис може допомогти прояснити ситуацію.

Декларація виглядає так:

.box {
    animation-timing-function: linear;
}

Ця властивість приймає наступні значення ключових слів:

  • ease (початкове значення)
  • ease-in
  • ease-out
  • ease-in-out
  • linear
  • step-start
  • step-end

Більшість значень відносно легко зрозуміти за їхніми назвами, але ви можете побачити, чим вони відрізняються на Codepen

За допомогою клавіші вибору виберіть одне з семи значень ключового слова. Зверніть увагу, що значення ease-* "полегшують" анімацію у різний спосіб. Лінійне значення є послідовним на всьому шляху.

Властивість animation-timing-function також приймає наступні функції:

  • cubic-bezier(): приймає в якості аргументів чотири числові значення, відокремлені комами
  • steps(): приймає як аргументи два значення, число та "член переходу", розділені комою

Значення ключових слів step-start і step-end еквівалентні значенням steps(1, jump-start) і steps(1, jump-end) відповідно.

Що стосується cubic-bezier(), то ви можете зрозуміти цю функцію трохи краще на Codepen

Зверніть увагу, що демонстрація дозволяє вам задавати чотири аргументи у функції cubic-bezier(). Два аргументи можуть бути від'ємними, а два обмежені десятковими значеннями від 0 до 1.

Функція steps(), з іншого боку, приймає два аргументи:

  • Ціле число, що представляє рівні "зупинки" протягом одного циклу анімації.
  • Необов'язкове ключове слово "jumpterm", яке визначає, чи анімація "затримується" на певних інтервалах

Знову ж таки, ви можете зрозуміти як працює функція на Codepen

animation-iteration-count

У деяких випадках вам буде достатньо одного запуску анімації, але іноді ви хочете, щоб анімацію було запущено декілька разів. Властивість animation-iteration-count дозволяє вам зробити це, прийнявши додатне число, що представляє кількість разів, яку ви хочете, щоб анімація була запущена:

.box {
    animation-iteration-count: 3;
}

Початковим значенням для параметра animation-iteration-count є 1, але ви також можете використати ключове слово infinite (не потребує пояснень) або використати дробове значення. Дробове значення призведе до запуску анімації на половині шляху на дробовому прогоні:

.box {
    animation-iteration-count: 3.5;
}

Вищенаведений код прокручує анімацію три з половиною рази. Тобто, три повні ітерації, за якими слідує одна фінальна ітерація, яка зупиняється рівно на півдорозі.

Ця властивість є найбільш корисною у поєднанні з властивістю animation-direction (розглянутою далі), оскільки анімація, яка запускається лише з самого початку, не є надзвичайно корисною, якщо вона запускається більше одного разу.

Ви можете спробувати демонстрацію, яка дозволяє вибрати дробове значення для кількості ітерацій, щоб побачити ефект на Codepen

animation-direction

Як згадувалося вище в цьому підручнику з CSS-анімації, властивість animation-direction чудово працює в поєднанні з animation-iteration-count. Властивість animation-direction дозволяє вам визначити напрямок відтворення анімації. Синтаксис виглядає наступним чином:

.box {
    animation-direction: alternate;
}

Ви можете встановити значення як одне з чотирьох ключових слів:

  • normal: анімація відтворюється вперед на першій ітерації (за замовчуванням)
  • reverse: анімація відтворюється у зворотному порядку на першій ітерації
  • alternate: анімація відтворюється вперед на першій ітерації, а потім чергується на наступних ітераціях
  • alternate-reverse: те саме, що й alternate, але відтворюється у зворотному порядку на першій ітерації

Ви можете спробувати різні значення з різною кількістю ітерацій за допомогою інтерактивної демонстрації на Codepen

animation-play-state

Властивість animation-play-state не дуже корисна у статичному середовищі CSS, але може стати в нагоді при написанні інтерактивних анімацій за допомогою JavaScript або навіть CSS.

Ця властивість приймає два значення: running або paused:

.box {
    animation-play-state: paused;
}

За замовчуванням, будь-яка анімація перебуває у стані "запущена". Але ви можете використовувати JavaScript, щоб змінити значення властивості. Ви навіть можете використовувати інтерактивні CSS-функції, такі як :hover або :focus, щоб змінити стан анімації на "призупинений", що, по суті, заморозить анімацію, де б вона не знаходилася в поточній ітерації.

Інтерактивна демонстрація на Codepen має нескінченну анімацію з двома кнопками для "призупинення" та "відновлення" анімації.

animation-delay

Деякі анімації призначені для негайного початку анімації, тоді як інші можуть виграти від невеликої затримки перед першою ітерацією. Властивість animation-delay дозволяє це зробити.

.box {
    animation-delay: 4s;
}

Як і інші значення, що базуються на часі, ви можете встановити animation-delay в секундах або мілісекундах. Ви також можете використовувати дробові значення.

Важливо зазначити, що затримка відбувається лише на першій ітерації, а не на кожній наступній. Ви можете спробувати це за допомогою інтерактивної демонстрації на Codepen

У демонстраційній версії ви можете змінювати значення ітерації, а також затримку, тому ви можете побачити, що затримка не впливає на наступні ітерації - тільки на першу.

Цікавим варіантом використання цієї властивості є від'ємне значення. Наприклад, використовуючи демонстрацію вище, спробуйте встановити animation-delay на -0.5s. Ви помітите, що від'ємна затримка працює майже так само, як рух вперед у машині часу - анімація починається на півдорозі, а не на початку.

animation-fill-mode

Останньою властивістю longhand, яку я розгляну в цьому підручнику з CSS-анімації, є та, яку я використовував у попередній демонстрації. Ви помітите, що коли анімація зупиняє останню ітерацію, поле залишається на місці. Це досягається за допомогою animation-fill-mode.

.box {
    animation-fill-mode: forwards;
}

Ця властивість визначає, як анімація застосовує стилі до цільового елемента до і після виконання. Це трохи складно зрозуміти концептуально, тому я поясню значення кожного значення, а потім ви зможете порівняти їх за допомогою інтерактивної демонстрації.

Ця властивість приймає наступні чотири значення ключових слів:

  • none: стилі не застосовуються ні до, ні після виконання (за замовчуванням)
  • forwards: зберігає всі стилі, застосовані в останньому ключовому кадрі анімації
  • backwards: більш-менш протилежне попередньому значенню, воно зберігає стилі, застосовані до анімації, як тільки вона починає виконуватися, але до початку анімації
  • both: Зберігає стилі як для forwards, так і для backwards

Фінальна демонстрація на Codepen в цьому посібнику з CSS-анімації зробить все трохи зрозумілішим, але може знадобитися багато часу, перш ніж ви дійсно зрозумієте, що вона робить і як цього досягти.

animation

Після того, як ви добре зрозумієте кожну з властивостей, у вас буде можливість скористатися властивістю скорочення animation, яка дозволяє вам визначити всі повні властивості в одній декларації.

.box {
    animation: moveObject 2s ease-in-out 3 reverse running 1.3s forwards;
}

Чесно кажучи, це дуже багато інформації в одному рядку. Я б рекомендував використовувати скорочення, якщо декларація включає не всі властивості, а лише три-чотири з них.

Якщо ви використовуєте скорочення, ви можете розташовувати значення в будь-якому порядку, але дотримуйтесь наступних правил:

  • Перше значення, яке визначає час, - це animation-duration; будь-яке наступне значення часу - це animation-delay.
  • Якщо існує конфлікт між ключовим словом і animation-name, значення ключового слова матиме пріоритет, якщо воно з'явиться першим.
  • Будь-які пропущені значення повернуться до початкового стану, як і у випадку з будь-якою властивістю скорочення CSS.

Застосування декількох анімацій до одного елемента

Ще одна особливість, про яку варто знати, полягає в тому, що ви можете застосувати кілька анімацій до одного об'єкта, розділяючи їх комами.

Ось приклад з використанням скорочення:

.box {
    animation: moveObject 2s ease-in-out, fadeBox 3s linear;
}

Тут я визначив дві різні анімації, які відповідають двом різним наборам ключових кадрів, але застосовуються до одного й того ж об'єкта. Цей самий код можна написати від руки так:

.box {
    animation-name: moveObject, fadeBox;
    animation-duation: 2s, 3s;
    animation-timing-function: ease-in-out, linear;
}

Зверніть увагу, що кожна властивість містить два значення, розділених комами. У цьому випадку скорочений запис майже напевно буде легше читати і підтримувати.
