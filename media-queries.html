Медіа-запити

Підтримуйте однаковий рівень користувацького досвіду для різних пристроїв.
Змінюйте вигляд сторінки в залежності від розміру екрану.

Коротко про Медіа-запити

CSS Медіа-запити - @media - дозволяють стилізувати кілька версій одного сайту залежно від типу пристрою, на якому переглядається макет/сайт. Іншими словами, це конструкції, які дозволяють вказати, виходячи з певних умов, які стилі слід використовувати на веб-сторінці, а які ні.

Мета медіа-запитів - допомогти розробникам створювати адаптивні макети. Адаптивний макет означає, що він адаптується до ширини екрану пристрою.

Усі адаптивні макети починаються на рівні області перегляду, а саме метатегу області перегляду. Цей тег повідомляє браузеру, що ширина видимої області макета повинна дорівнювати ширині пристрою.

Це значення не залежить від фактичної роздільної здатності екрана; натомість воно призначене для того, щоб адаптивний дизайн веб-сайту відображався однаково на всіх пристроях, незалежно від щільності пікселів на екрані. Іншими словами, він зіставляє CSS з фізичною роздільною здатністю веб-сторінки.

<meta name="viewport" content="width=device-width, initial-scale=1">

Ось ключові відмінності між Viewport та Медіа-запитами:

  • При налаштуванні адаптивності через область перегляду ми можемо налаштувати стиль відповідно до області перегляду пристрою. Стилі за допомогою CSS залишаються незмінними; область перегляду не має жодних змін у дизайні.
  • Медіа-запити, однак, мають іншу функцію. Вони допомагають налаштувати наш стиль відповідно до обмежень, встановлених розміром екрану. Зміни дизайну за допомогою медіа-запитів є кастомними і відображаються залежно від поточного розміру екрану.

Синтаксис @media

Створення медіа-запиту починається з ключового слова @media, за яким слідує одна або кілька умов. Умовою може бути тип пристрою або вимога до певної характеристики пристрою. Вимога до певної характеристики записується в круглих дужках.

Об'єднання декількох умов здійснюється за допомогою логічних операторів.

Після створення запиту @media зазначені в ньому стилі будуть застосовані лише тоді, коли кінцевий результат функції повернеться як true.

Ось приклад медіа-запиту з однією умовою:

@media screen {
    /* стиль буде застосовано до будь-якого пристрою з "екраном" */.
}

/* це дозволяє встановити інший шрифт для @media print - наприклад */.
@media screen {
    * { font-family: Georgia; }
}

Ось приклад медіа-запиту з кількома умовами:

@media (min-width: 570px) and (max-width: 768px) {
    ...
}

У наведеному вище прикладі стиль буде застосовано лише до пристрою, який відповідає параметрам мінімальної ширини області перегляду 570 пікселів і максимальної - 768 пікселів.

Використання медіа-запитів поза CSS

Медіа-запити - це властивість CSS, але вони часто використовуються як з JavaScript, так і з HTML-документами, коли використовуються елементи <link> для визначення таблиць стилів.

Якщо мова йде про HTML, щоразу, коли ви використовуєте елемент <link> для посилання на таблицю стилів, ви можете додати атрибут media="...", який підтримує фактичні медіа-властивості CSS.

/* example */

<link href="mobile.css" rel="stylesheet" media="screen and (max-width: 600px)">

/* detect OS theme preference directly */

<link href="css/light.css" rel="stylesheet" media="(prefers-color-scheme: light)">
<link href="css/dark.css" rel="stylesheet" media="(prefers-color-scheme: dark)">

Типи пристроїв для медіа-запитів

Існує чотири типи пристроїв, які ми можемо вказати:

  • all: медіа-вираз буде застосовано до всіх пристроїв. Якщо ви не вкажете жодного типу, за замовчуванням буде застосовано цей.
  • print: стилі в межах цього медіа-виразу застосовуються під час друку на принтері або експорту до PDF, у тому числі в режимі попереднього перегляду документа.
  • screen: для пристроїв з екраном.
  • speech: для мовних синтезаторів (поки що не підтримується жодним браузером).

У більшості випадків, коли ви пишете стилі тільки для екрану, вам не потрібно вказувати screen або all. Реальне практичне застосування має лише тип друку.

Окрім типів пристроїв, існують також їхні характеристики. Вони можуть варіюватися від простих налаштувань на основі пристрою до специфічних налаштувань користувача, таких як "віддає перевагу темним кольорам".

Логічні оператори

При написанні медіа-запитів можна використовувати чотири різні логічні оператори:

  • ,
  • and
  • not
  • only (застарілий)

and

Цей оператор використовується для об'єднання декількох умов. У цьому випадку їхня логіка буде виконана, коли кожна з них буде true.

Наприклад, наступний запит @media буде застосовано, тільки якщо виконано всі три умови (screen, width >= 1280px і орієнтація landscape):

@media screen and (min-width: 1280px) and (orientation: landscape) {
    ...
}

, (кома)

Цей оператор визначає використання стилів, коли потрібно виконати лише одну із зазначених умов.

У цьому прикладі стилі будуть застосовані до сторінки у двох випадках: width >= 524px або орієнтація portrait.

@media (min-width: 524px), (orientation: portrait) {
    ...
}

not

Оператор not використовується для заперечення.

При використанні not з and заперечення працює для всього медіа-запиту. У цьому випадку при вказівці not - не потрібно вказувати тип пристрою.

Наприклад, застосовувати стилі лише тоді, коли це не (screen і width >= 418px і height >= 724px).

@media not screen and (min-width: 418px) and (min-height: 724px) {
    ...
}

При використанні not у виразі з комами додається заперечення лише для цієї частини.

Наприклад, застосовувати стилі, коли виконується наступна умова: not a screen or not width >= 418px.

@media not screen, not (min-width: 418px) {
    ...
}

Особливості (характеристики) медіа-запитів

Цей розділ продовжить розділ про типи пристроїв, який ми розглядали раніше. Багато з "характерних вимог" або "медіа-функцій" є дуже специфічними і багато в чому непрактичними для повсякденної роботи розробника. Замість цього, цей розділ зосередиться на найпопулярніших типах медіа-запитів, які корисні в повсякденній роботі з CSS.

width

Характеристика width медіа дозволяє вам задати, щоб ширина області перегляду дорівнювала певному значенню.

Наприклад, застосувати CSS лише до області перегляду з width: 316px;.

@media (width: 316px) { ... }

Ви можете використовувати min-width, а потім визначити діапазон з max-width.

Наприклад, @media для ширини області перегляду від 586px до 1280px:

@media (min-width: 586px) and (max-width: 1279.98px) { ... }

/* Іншими словами, цей запит не буде працювати для ширини більше 1280px. */

Для ширини більше 768px:

@media (min-width: 768px) { ... }

Якщо вам потрібно менше 1536px:

@media (max-width: 1535.98px) { ... }

height

Ви можете використовувати height та min-height, щоб задати умови для висоти області перегляду.

Наприклад, @media для висоти області перегляду більше 720px:

@media (min-height: 720px) { ... }

orientation

За допомогою orientation можна встановити певні стилі залежно від того, в якому режимі (альбомному чи книжковому) відображається сайт.

Наприклад, залежно від орієнтації вікна перегляду ви можете відображати різні зображення:

@media (orientation: landscape) {
    .featured-hero { background: url(hero-l.png) no-repeat; }
}

@media (orientation: portrait) {
    .featured-hero { background: url(hero-p.png) no-repeat; }
}

aspect-ratio

Характеристики aspect-ratio, min-aspect-ratio, max-aspect-ratio дають змогу встановлювати стилі залежно від співвідношення сторін у вікні перегляду.

/* Мінімум */
@media (min-aspect-ratio: 9/16) {
    .featured-hero { background-color: blue; }
}

/* Максимум */
@media (max-aspect-ratio: 16/9) {
    .featured-hero { background-color: red; }
}

/* Точно */
@media (aspect-ratio: 1/1) {
    .featured-hero { background-color: white; }
}

resolution

Характеристики resolution, min-resolution та max-resolution можна використовувати, коли ви хочете встановити стилі залежно від щільності пікселів пристрою.

Наприклад, встановити інший розмір шрифту для пристроїв з щільністю пікселів на дюйм більше 150:

/* За замовчуванням */
h2  {
    font-size: 1.5rem;
}

/* Мінімальна роздільна здатність */
@media (min-resolution: 150dpi) {
    h2  {
        font-size: 1.1rem;
    }
}
