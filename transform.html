Трансформації

3D-трансформації, зміни розміру, поворот та пересування елементів на сторінці.
Створюйте різні елементи, які змінюються за допомогою відмінних від стандартних трансформацій.

Коротко про Трансформації

Властивість transform дозволяє візуально маніпулювати елементом, нахиляючи, обертаючи, перекладаючи або масштабуючи його:

.element {
    width: 20px;
    height: 20px;
    transform: scale(20);
}

Навіть із заявленими висотою і шириною, цей елемент тепер буде масштабовано в двадцять разів більше, ніж його початковий розмір:

Якщо надати цій функції два значення, то вона розтягнеться по горизонталі на перше значення, а по вертикалі - на друге. У наступному прикладі елемент тепер буде вдвічі ширшим, але вдвічі меншим за висоту від початкового елемента:

.element {
    transform: scale(2, .5);
}

Або ви можете бути більш конкретними, не використовуючи функцію скорочення:

transform: scaleX(2);
transform: scaleY(.5);

Але scale() - це лише одна з багатьох доступних функцій перетворення.

Значення

  • scale(): впливає на розмір елемента. Це також стосується font-size, padding, height та width елемента. Це також скорочена функція для функцій scaleX і scaleY.
  • skewX() та skewY(): нахиляє елемент вліво або вправо, наприклад, перетворюючи прямокутник на паралелограм. skew() - це скорочена функція, яка поєднує skewX() і skewY, приймаючи обидва значення.
  • translate(): переміщує елемент убік або вгору і вниз.
  • rotate(): повертає елемент за годинниковою стрілкою з поточного положення.
  • matrix(): функція, яка, ймовірно, не призначена для написання вручну, але об'єднує всі перетворення в одне.
  • perspective(): не впливає на сам елемент, але впливає на трансформації 3D-трансформацій нащадків, дозволяючи їм усім мати узгоджену перспективу глибини.

skew

/* Точки нахилу вздовж осі x */
.element {
    transform: skewX(25deg);
}

/* Точки нахилу вздовж осі y */
.element {
    transform: skewY(25deg);
}

/* Точки нахилу вздовж осей x та y */
.element {
    transform: skew(25deg, 25deg);
}

Функції перетворення skewX і skewY нахиляють елемент у той чи інший бік. Пам'ятайте: для нахилу елемента не існує властивості скорочення, тому вам доведеться використовувати обидві функції. У наведеному на Codepen прикладі ми можемо нахилити квадрат розміром 100px x 100px вліво і вправо за допомогою skewX.

У цьому на Codepen прикладі ми можемо нахилити елемент по вертикалі за допомогою skewY.

Тепер використаємо skew(), щоб об'єднати їх.

rotate

.element {
    transform: rotate(25deg);
}

Це обертає елемент за годинниковою стрілкою від його початкової позиції, тоді як від'ємне значення обертає його в протилежному напрямку. Простий анімований приклад на Codepen, де квадрат продовжує обертатися на 360 градусів кожні три секунди.

translate

.element {
    transform: translate(20px, 10px);
}

Ця функція перетворення переміщує елемент убік або вгору і вниз. Чому б просто не використати верхній/лівий/нижній/правий? Ну, іноді це трохи заплутано. Можна вважати це компонуванням/позиціонуванням (вони все одно краще підтримуються браузерами), а це - способом переміщення цих елементів в рамках переходу або анімації.

Це можуть бути будь-які значення довжини, наприклад, 10px або 2.4em. Одне значення перемістить елемент праворуч (від'ємні значення - ліворуч). Якщо вказати друге значення, то воно перемістить його вниз (від'ємні значення вгору). Або ви можете бути більш конкретними:

    transform: translateX(value);
    transform: translateY(value);

Важливо зазначити, що елемент, який використовує transform, не спричиняє обтікання інших елементів. Скориставшись функцією translate нижче і виштовхнувши зелений квадрат з його початкового положення, ми побачимо на цьому прикладі на Codepen, що навколишній текст залишиться фіксованим на місці, так, ніби квадрат є блоковим елементом.

Варто також зазначити, що translate буде апаратно прискорено, якщо ви захочете анімувати цю властивість, на відміну від position: absolute;.

Кілька значень transform

За допомогою списку, розділеного пробілами, ви можете додати кілька значень до властивості transform:

.element {
    width: 20px;
    height: 20px;
    transform: scale(20) skew(-20deg);
}

Варто зазначити, що існує порядок, в якому ці перетворення будуть виконуватися, у наведеному вище прикладі спочатку буде виконано "нахил", а потім елемент буде масштабовано.

matrix

Функція матричного перетворення може бути використана для об'єднання всіх перетворень в одне. Це схоже на скорочення перетворень, тільки напевно вона призначена для написання вручну. Існують інструменти, такі як The Matrix Resolutions, які можуть перетворити групу перетворень в одне оголошення матриці. Можливо, у деяких ситуаціях це може зменшити розмір файлу, хоча такі недружні до автора мікрооптимізації, швидше за все, не варті вашого часу.

Для допитливих - ось це:

rotate(45deg) translate(24px, 25px)

...можна також представити як:

matrix(0.7071067811865475, 0.7071067811865476, -0.7071067811865476, 0.7071067811865475, -0.7071067811865497, 34.648232278140824)

3D-трансформації

Більшість з перерахованих вище властивостей мають 3D-версії.

translate3d(x, y, z)
translateZ(z)

Третє значення в translate3d або значення в translateZ переміщує елемент до глядача, від'ємні значення - від нього.

scale3d(sx, sy, sz)
scaleZ(sz)

Третє значення у scale3d або значення у scaleZ впливає на масштабування вздовж осі z (наприклад, уявна лінія, що виходить прямо з екрана).

rotateX(value)
rotateY(value)
rotate3d(x, y, z)

rotateX і rotateY обертають елемент у тривимірному просторі навколо цих осей. rotate3d дозволяє вказати точку у тривимірному просторі, навколо якої обертатиметься елемент.

matrix3d(…)

Спосіб програмно описати 3D-трансформацію в сітці 4×4. Ніхто ніколи не напише таке вручну, ніколи.

perspective(value)

Це значення не впливає на сам елемент, але впливає на трансформації 3D-трансформацій нащадків, дозволяючи їм усім мати послідовну перспективу глибини.
